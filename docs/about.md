# О проекте

Проект представляет собой статический сайт, созданный с использованием генератора MkDocs и размещенный на платформе GitHub Pages.

## Цели работы

1. Изучить процесс создания статических сайтов с использованием Python-генераторов
2. Настроить автоматический деплой через GitHub Actions
3. Исследовать возможности GitHub Pages для хостинга
4. Изучить основы CI/CD на практике

## Выбор инструментов

### MkDocs

Для проекта был выбран генератор MkDocs по следующим причинам:
- Простота установки и настройки
- Хорошая документация
- Поддержка Markdown
- Наличие готовых тем оформления

### GitHub Pages

GitHub Pages был выбран как платформа хостинга благодаря:
- Бесплатному использованию для публичных репозиториев
- Автоматической интеграции с GitHub Actions
- Встроенной поддержке SSL
- Простоте настройки

## Процесс разработки

Разработка включала следующие этапы:

1. Настройка локального окружения (Python, Git)
2. Создание структуры проекта и установка зависимостей
3. Разработка контента в формате Markdown
4. Локальное тестирование с помощью встроенного сервера
5. Настройка автоматического деплоя через GitHub Actions
6. Публикация на GitHub Pages

## Архитектура

Сайт построен по следующей схеме:
- Контент хранится в файлах `.md` в папке `docs/`
- Конфигурация описана в файле `mkdocs.yml`
- При изменениях в ветке main автоматически запускается процесс сборки
- Собранный сайт публикуется в ветке gh-pages
- GitHub Pages автоматически обновляет опубликованную версию
