    Отчет о выполнении задания¶

    1. Подготовительный этап¶

    Проверка установленного ПО¶

    Была выполнена проверка наличия необходимых инструментов:

    python --version  # Python 3.11.2
git --version     # git version 2.46.2

    Создание структуры проекта¶

    Создан каталог проекта на рабочем столе и выполнена инициализация Git:

    mkdir PyWeb
cd PyWeb
git init

    Настройка виртуального окружения¶

    Создано и активировано виртуальное окружение Python:

    python -m venv venv
venv\Scripts\activate

    2. Установка и настройка MkDocs¶

    Установка пакетов¶

    Установлены необходимые библиотеки:

    pip install mkdocs mkdocs-material
pip freeze > requirements.txt

    Инициализация проекта¶

    Создана базовая структура MkDocs:

    mkdocs new .

    Команда создала файл конфигурации mkdocs.yml и каталог docs/ с начальным содержимым.

    3. Настройка репозитория GitHub¶

    Создание репозитория¶

    Создан публичный репозиторий: github.com/Adzhevelik/PyWeb

    Связывание с локальным проектом¶

    Выполнено подключение удаленного репозитория:

    git remote add origin https://github.com/Adzhevelik/PyWeb.git
git pull origin main --allow-unrelated-histories

    4. Разработка контента¶

    Настройка конфигурации¶

    В файле mkdocs.yml настроены: - Информация о сайте - Тема Material - Структура навигации - Расширения Markdown

    Создание страниц¶

    Разработаны следующие страницы: - index.md - главная страница - about.md - описание проекта - report.md - отчет о выполнении

    Локальное тестирование¶

    Выполнена проверка работы сайта локально:

    mkdocs serve

    Сайт был доступен по адресу http://127.0.0.1:8000/

    5. Настройка CI/CD¶

    Создание workflow¶

    Создан файл .github/workflows/deploy.yml со следующей конфигурацией: - Триггер: push в ветку main - Шаги: установка Python, установка зависимостей, сборка сайта, деплой

    Процесс автоматизации¶

    При каждом изменении в ветке main: 1. GitHub Actions запускает workflow 2. Устанавливается окружение и зависимости 3. Выполняется команда mkdocs build 4. Результат публикуется в ветку gh-pages

    6. Публикация на GitHub Pages¶

    Активация GitHub Pages¶

    В настройках репозитория включен GitHub Pages: - Source: Deploy from a branch - Branch: gh-pages - Folder: / (root)

    Результат¶

    Сайт опубликован и доступен по адресу: https://adzhevelik.github.io/PyWeb/

    7. Исследование дополнительных вопросов¶

    7.1 Российские CDN-провайдеры¶

    Изучены следующие варианты CDN для ускорения доставки контента:

    Yandex Cloud CDN - Интеграция с Yandex Object Storage - Распределенная сеть серверов - Гибкая тарификация

    Selectel CDN - Более 60 точек присутствия в России - Поддержка HTTPS - API для управления

    VK Cloud CDN - Часть облачной платформы VK Cloud - Интеграция с другими сервисами платформы

    7.2 GitVerse для CI/CD¶

    GitVerse - российская платформа для хостинга Git-репозиториев.

    Основные возможности: - Хостинг репозиториев с хранением данных в РФ - Встроенная система CI/CD через файлы .gitverse-ci.yml - Интеграция с российскими облачными провайдерами - Техподдержка на русском языке

    Пример конфигурации CI/CD:

    stages:
  - build
  - deploy

build_site:
  stage: build
  script:
    - pip install -r requirements.txt
    - mkdocs build

    7.3 Варианты деплоя статических сайтов¶

    GitHub Pages - Преимущества: бесплатно, простая настройка, SSL - Недостатки: ограничение на размер, только статика

    Yandex Object Storage - Преимущества: масштабируемость, CDN, российская юрисдикция - Недостатки: платный сервис, требуется настройка

    Собственный VPS - Преимущества: полный контроль, гибкость настройки - Недостатки: требуется администрирование, настройка безопасности

    Netlify / Vercel - Преимущества: удобный интерфейс, много функций - Недостатки: зарубежные сервисы

    Необходимые инструменты для деплоя: - Git для контроля версий - SSH для доступа к серверу (при использовании VPS) - Nginx/Apache для веб-сервера - Certbot для SSL-сертификатов - AWS CLI или аналоги для работы с Object Storage
